MATLAB interface via \ref gsCInterface

The MATLAB interface of G+Smo is enabled via the C interface of G+Smo. The package is available as the gsMatlab module.

Installation of gsMatlab

Calling the C interface in MATLAB

Inside MATLAB, the G+Smo class hierarchy is maintained as much as possible. The gsFunctionSet-equivalent in MATLAB is defined as:

classdef gsFunctionSet < handle
% Class implementations
end

And, for example, a gsMultiPatch deriving from gsFunctionSet is declared as:

classdef gsMultiPatch < gsFunctionSet
% Class implementations
end

Within the classes, member functions can be declared, which call their corresponding function from the C interface of G+Smo:

function varargout = addPatch(this, varargin)
if (nargin~=2 || nargout>0)
error('Invalid number of input and/or output arguments.')
end
calllib('gismo','gsMultiPatch_addPatch',this.objectHandle,varargin{1}.ptr());
end
gismo
The G+Smo namespace, containing all definitions for the library.
Note
Since MATLAB does not natively offer an interface to Eigen, the gsMatrix is binded to MATLAB using the EigenMatrix types. This type can easily convert to a MATLAB matrix and can be given to the C interface of G+Smo.

Further information and examples

For further information and examples regarding the MATLAB interface, we refer to the gsMatlab repository.
