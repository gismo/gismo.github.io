gsCApi

This is the submodule that the Lisp ninjas use for their magic and where other C-interfaced activities take place.

Three basic rules

  1. The code in gsCApi.h, gsPKInterface.cpp and some functions from gsCInterface.h (see below) are in productive use at MTU. Therefore it is important that their outward behaviour does not change.
  2. This is not open-source but rather code developed at MTU that friends may use. While the contents is not awfully critical, MTU still needs to be in control of where the code spreads.
  3. We can talk anything through in a collaborative way.(-; But please, first a discussion and only then breaking changes.

The functions from gsCInterface.h that are used by MTU: gs_delete_array, gs_load_bspline, gs_eval_geometry, gs_delete_bspline and gs_load_thb_spline.

Components

The source code consists of three main components:

  • C-interface for easy linking,
  • several specialised algorithms on top of G+Smo accessible through the interface,
  • supporting functions that used to be in unsupported and that are required by these algorithms. Currently, it is only gsHFittingLvlConstrained.h.

Dependencies

The full module depends on the following G+Smo modules:

  • gsParasolid,
  • gsUnstructuredSplines: the gsMPBES-classes are the development of Florian's gsSmoothPatches that we use for periodic fitting,
  • gsAdaptiveFitting: currently only gsScatteredHFitting but we intend to expand in this direction.

Getting started

Currently, it is necessary specify the dependencies manually, otherwise one ends up with a confusing error message.

> git checkout https://github.com/gismo/gismo
> mkdir release
> mkdir install
> cd release
> cmake ../gismo/ -DGISMO_BUILD_EXAMPLES=OFF -DGISMO_OPTIONAL="gsUnstructuredSplines;gsParasolid;gsAdaptiveFitting;gsCApi" -DPARASOLID_INCLUDE_DIR=/path/to/parasolid -DPARASOLID_LIBRARY=/path/to/parasolid/shared_object/libpskernel.so -DCMAKE_INSTALL_PREFIX=/where/to/install
> make -j7
> make install

The shared object should now be in /where/to/install/lib and can be linked as required. This capability can be also tested by calling

> nm libgismo.so | grep "gs_bspline_periodic_fitting"

and checking that there is T before the function name.

Pitfalls

If you call

> make gsCApi -j7

the make step will be surprisingly fast. However, it does not change the shared object, potentially leading to confusion.
